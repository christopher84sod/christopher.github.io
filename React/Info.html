Ramverk & Bibliotek

Det finns ramverk som existerar idag som Vue, Angular och Ember som kan hantera vyerna, men React klassas inte som ett ramverk utan är egentligen ett Javascript bibliotek.

Skillnaden mellan ramverk och bibliotek, om man tar Angular till exempel, så är det ett ramverk med tydliga strukturer och flera färdiga, inbyggda funktioner, och fungerar lite mer som en helhetslösning, React och andra sidan, är ett bibliotek som du lätt kan implementera många olika typer av paket, med i stort sett oändliga möjligheter när det kommer till anpassning. Bygger man ett större projekt i React ha in andra bibliotek. Och glöm inte att React stöds inte i någon webbläsare äldre än IE8.

React använder ett virtuellt DOM som är som en ritning av DOM:et. När förändringar sker så uppdateras inte hela DOM:et utan istället enbart de delarna som har förändras. Viktigt att veta är att React inte tillhandahåller stöd för någon server-side (back-end) hantering utan endast inom det som sker i webbläsaren (front-end). Detta betyder att val av server-side hantering är fritt i kombination med React

Än så länge så kan man inte enbart använda React i större projekt utan man måste ha in andra bibliotek . Det tar en tag att lära sig så allt arbete i början tar ganska lång tid. Och glöm inte att React stöds inte i någon webbläsare äldre än IE8.
